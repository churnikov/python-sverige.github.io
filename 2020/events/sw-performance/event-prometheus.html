Writing Prometheus Exporters in Python

Video (Q&A on Slack): https://youtu.be/SOgAEXzLWxA

Abstract: Prometheus is quickly becoming the defacto standard for metrics collection. For those services which do not support Prometheus natively, it is possible to write an exporter: something that will let prometheus monitor it. I will briefly cover the use cases and set-up, with most of the session being practical parts of how to write an exporter.

About the speaker: Moshe Zadka has been involved in the Linux community since 1998, helping in Linux "installation parties". He has been programming Python since 1999, and has contributed to the core Python interpreter. Moshe has been a DevOps/SRE since before those terms existed, caring deeply about software reliability, build reproducibility and other such things. He has worked in companies as small as three people and as big as tens of thousands -- usually some place around where software meets system administration.
